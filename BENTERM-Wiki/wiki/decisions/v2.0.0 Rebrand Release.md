# v2.0.0 Rebrand Release

## What

A pure-visual rebrand pass on top of the v1.4.5 engine. No functional changes — same attention system, same OSC 9 trigger, same shell integration, same config schema. The release replaces every user-facing surface that still carried BenjaminTerm or upstream WezTerm visual brand with the BENTERM identity (BEN red graphic, "BENTERM" wordmark, retro-CRT banner aesthetic).

Tag: `v2.0.0`. Released: 2026-04-28. Force-pushed over the original v2.0.0 (which had been the brand-rename ceremony commit only).

## Scope of the Rebrand

### Visual surfaces (all rebranded)

- **GitHub README banner** (`assets/banner.webp`, 1915×821 WebP).
- **GitHub social preview** (`assets/github-social-preview.jpg`, 1280×640 JPEG).
- **Windows icon** (`assets/windows/terminal.ico`, 7 sizes 16-256), embedded in `benterm-gui.exe` and used by the Inno Setup installer.
- **macOS icon** (`assets/macos/BENTERM.app/Contents/Resources/terminal.icns`, 7 sizes 16-1024).
- **Linux icon** (`assets/icon/terminal.png`, 128×128).
- **Source for all platform icons**: `assets/icon/BENTERM.jpg` (1024×1024 BEN red graphic).

### File renames

- `assets/macos/WezTerm.app/` → `assets/macos/BENTERM.app/`.
- `assets/wezterm.desktop` → `assets/benterm.desktop`.
- `assets/wezterm.appdata.xml` → `assets/benterm.appdata.xml`.
- `assets/wezterm-nautilus.py` → `assets/benterm-nautilus.py`.
- `assets/open-wezterm-here` → `assets/open-benterm-here`.
- `assets/flatpak/org.wezfurlong.wezterm.*` → `assets/flatpak/com.avalonreset.benterm.*` (json + xml).

### Identity strings

- Linux desktop entry: `Name=BENTERM`, `Icon=com.avalonreset.benterm`, `TryExec=benterm`, `Exec=benterm start --cwd .`.
- Linux AppStream id: `com.avalonreset.benterm`.
- Linux launchable id: `com.avalonreset.benterm.desktop`.
- macOS bundle id: `com.avalonreset.benterm` (already correct in Info.plist).
- macOS `CFBundleExecutable`: `benterm-gui` (lowercase, fixed from `BENTERM-gui` to match what the package script installs).
- Bundled distro Lua config name: `benterm.lua` (was `wezterm.lua`). All four shippers (Inno Setup, Windows portable zip, macOS app bundle, Linux tarball) updated.

### Deletions

- `assets/icon/wezterm-icon.svg` and `assets/icon/wezterm-ghifarit53-{1,2,3}.svg` (legacy upstream SVG icon set).
- `assets/icon/benterm_screenshot.{jpg,webp}` (had "WEZTERM-POWERED" baked into the screenshot image).

### Doc softening

- README: dropped the "Built from WezTerm" badge and the "WezTerm (vanilla)" comparison column. Banner alt text de-WezTerm'd. Upstream credit retained in its dedicated section per MIT requirements.

### CI updates

- 30+ `gen_*.yml` workflow trigger paths updated to track the renamed files.
- `ci/generate-workflows.py` `TRIGGER_PATHS_UNIX` updated.
- `ci/windows-installer.iss`, `ci/package-benterm-windows.ps1`, `ci/package-benterm-macos.sh`, `.github/workflows/benterm-release.yml` (Linux step) all updated to ship config as `benterm.lua`.
- `wezterm-gui/build.rs:165` updated from `WezTerm.app` to `BENTERM.app` for the macOS-only `UNUserNotificationCenter` plist copy. Caught after first v2.0.0 push macOS-failed; landed in commit `4ccbe04a6`.

## What Was Intentionally NOT Renamed

These are internal namespacing references kept on `wezterm` / `org.wezfurlong.wezterm` because tools target them:

- Cargo workspace package names (`wezterm`, `wezterm-gui`, `wezterm-mux-server`, `strip-ansi-escapes`). Renamed at install time only.
- Window class `org.wezfurlong.wezterm` (registered in `wezterm-gui/src/lib.rs`). Hyper Yap targets this via window-class detection. Rename would break interop with all currently-installed Hyper Yap copies.
- Linux `.desktop` `StartupWMClass=org.wezfurlong.wezterm` to match the unchanged window class.
- Mux server socket prefix (`Local\wezterm-sock-*` on Windows, `wezterm-mux-server` on Unix).
- `%LOCALAPPDATA%\wezterm` data directory (used for blob leases, GPU cache, etc.).
- Terminfo entry name (`wezterm`).

[[Release Workflow Tax]] flags these for v3.0.0 consideration once Hyper Yap can ship a parallel update.

## Build Tooling Added

- `scripts/regenerate-banner.py` — Python stdlib-only KIE.ai gpt-image-2 client. Submits two text-to-image tasks (banner + social preview) using the BENTERM aesthetic prompt, polls for completion, downloads, and post-processes via ImageMagick to the target dimensions. Reusable for future banner refreshes.
- `assets/icon/update.sh` — points at `BENTERM.jpg` as the master source. Linux + macOS regeneration uses ImageMagick + `png2icns`. On a Windows dev box where `png2icns` is unavailable, Pillow can substitute (one-line Python: `Image.open(src).save(out, format='icns', sizes=[(16,16),(32,32),(64,64),(128,128),(256,256),(512,512),(1024,1024)])`).
