# Wiki Maintenance Flow

## For Every Meaningful Change

1. Read [[hot]].
2. Check [[index]] for existing pages.
3. Update the smallest relevant page.
4. Add new source pages only when the source is durable.
5. Update [[hot]] with current context.
6. Add a newest-first entry to [[log]].
7. Run wikilink and JSON validation when changing structure or canvas files.

## When Creating a New Note

Use `_templates/` and choose the closest note type:

- Decision
- Module
- Question
- Release Check
- Source

## When Unsure

Put the uncertainty in [[Open Release Questions]] or create a small question note.
