# BenjaminTerm Overview

BenjaminTerm is a cross-platform, vibe-coding terminal distribution built on top of WezTerm. It keeps the upstream terminal engine and adds a sensory workflow layer for long coding sessions: smart copy behavior, idle glow cues, pane-aware notifications, soft randomized attention sounds, fresh theme rotation, BenjaminTerm branding, custom release artifacts, and practical quality-of-life improvements for coding sessions.

The current project focus is release readiness across Windows, Linux, and macOS. The official release target is GitHub release artifacts with checksums. Package-manager ecosystems are optional follow-up channels.

The public repository surface now also includes a maintained README banner asset at `assets/banner.webp`. That asset should stay in `21:9` WebP format and should only be replaced by an approved candidate promoted from the local banner workflow described in [[README Banner Asset Workflow]].

## Product Vision

BenjaminTerm is not meant to be a radically different terminal engine. Its value is that it makes multi-window coding sessions easier to navigate by giving each session subtle identity and attention cues.

- Idle text glow helps identify a pane that has gone quiet and is ready for input.
- Pane-aware notifications can bring focus back to the exact pane or window that asked for attention.
- Soft randomized attention sounds help terminal windows become recognizable during long sessions.
- Shuffle-bag theme rotation gives sessions fresh character without repeating themes too quickly.
- The overall effect is meant to support fast attention switching during AI-assisted coding work.

## Current Release Shape

- Windows: installer and portable zip release artifacts.
- Linux: distro packages, source archive, raw tarball, and AppImage paths.
- macOS: BenjaminTerm `.app` bundle zip generated by the macOS packaging workflow.
